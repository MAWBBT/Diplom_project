# Цифровой портал аспирантуры

Веб-система личных кабинетов для аспирантуры: аспиранты, научные руководители, администратор и администратор программы. Клиент на **React (Vite)** обращается к REST API на **Node.js + Express + Sequelize**; данные хранятся в **PostgreSQL**.

## Возможности

- **Аспирант**: профиль, расписание, оценки, сообщения, уведомления, кабинет (ИУП, вехи, документы, публикации, аттестации, выгрузка архива).
- **Профессор**: журнал, кабинет руководителя (ведомые, массовое согласование планов), сообщения.
- **Администратор**: пользователи, расширенный доступ к журналу и оценкам (по правилам API).
- **Администратор программы**: сводки, список аспирантов, просроченные вехи, конструктор справок.

Авторизация — **JWT** (токен в `localStorage`, заголовок `Authorization: Bearer …`).

## Структура проекта

```
Diplom_On_React/
├── Backend/          # Express API, модели Sequelize
│   ├── config/       # database.js, jwt.js
│   ├── models/
│   ├── routes/
│   └── scripts/init-db.js
└── Frontend/         # React + Vite (сборка в Frontend/build/)
    └── src/
```

Папка может отображаться как `frontend` на Windows из‑за регистра — для сервера важно наличие `index.html` в каталоге со сборкой (см. ниже).

## Требования

- **Node.js** 18+
- **PostgreSQL** 14+ (или совместимая версия)

## Настройка базы данных

1. Создайте БД, например `edu_database`.
2. В каталоге `Backend` создайте файл `.env` (при необходимости):

| Переменная     | Назначение              | Пример / по умолчанию        |
|----------------|-------------------------|------------------------------|
| `DB_HOST`      | хост PostgreSQL         | `localhost`                  |
| `DB_PORT`      | порт                    | `5432`                       |
| `DB_NAME`      | имя БД                  | `edu_database`               |
| `DB_USER`      | пользователь            | `postgres`                   |
| `DB_PASSWORD`  | пароль                  | `12345678` (как в `config`)  |
| `PORT`         | порт HTTP сервера       | `3000`                       |
| `JWT_SECRET`   | секрет подписи JWT      | задайте в продакшене         |
| `JWT_EXPIRES_IN` | срок жизни токена     | `7d`                         |
| `CORS_ORIGIN`  | разрешённые Origin (через запятую) | по умолчанию localhost:5173 и свой `PORT` |
| `FRONTEND_PATH`| путь к статике относительно `Backend` | опционально |

## Установка

```bash
cd Backend && npm install
cd ../Frontend && npm install
```

## Инициализация данных (демо)

Скрипт **пересоздаёт таблицы** (`sync(true)`) и заполняет демо-данные. Не запускайте на продакшене с реальными данными.

```bash
cd Backend
npm run init-db
```

После выполнения в консоли выводятся подсказки по тестовым логинам.

## Запуск в режиме разработки

Терминал 1 — API:

```bash
cd Backend
npm start
```

Сервер слушает `http://localhost:3000`. Проверка: `GET http://localhost:3000/api/health`.

Терминал 2 — фронтенд (прокси `/api` → `localhost:3000`):

```bash
cd Frontend
npm run dev
```

Откройте `http://localhost:5173`. Вход выполняется только по **логину и паролю**; роль определяется записью пользователя в БД.

## Сборка и продакшен

```bash
cd Frontend
npm run build
```

Статика попадает в `Frontend/build/`. Express при старте ищет фронтенд по путям вроде `../Frontend/build` и `../Frontend` (где есть `index.html`). При нестандартном расположении задайте `FRONTEND_PATH` в `.env`.

Опционально скрипт `npm run deploy` в `Frontend` копирует содержимое `build/` в корень каталога `Frontend` (удобно, если сервер отдаёт статику из родительской папки фронта).

## API (кратко)

Базовый префикс: `/api`.

| Префикс            | Назначение              |
|--------------------|-------------------------|
| `/api/auth`        | вход                    |
| `/api/profile`     | профиль                 |
| `/api/schedule`    | расписание              |
| `/api/grades`      | оценки                  |
| `/api/messages`    | сообщения               |
| `/api/notifications` | уведомления           |
| `/api/journal`     | журнал (роли по API)    |
| `/api/admin`       | администрирование       |
| `/api/postgraduate`| кабинет аспиранта       |
| `/api/supervisor`  | кабинет руководителя    |
| `/api/program-admin` | админ программы       |

Список точек входа также отдаёт `GET /api`.

## Тестовые учётные записи (после `init-db`)

| Роль              | Логин           | Пароль        |
|-------------------|-----------------|---------------|
| Аспирант          | `postgraduate1` | `password123` |
| Профессор         | `professor1`    | `password123` |
| Администратор     | `admin1`        | `admin123`    |
| Админ программы   | `programadmin1` | `password123` |

Дополнительные пользователи (`postgraduate2` …, `professor2` …) создаются тем же скриптом.

| Команда              | Где        | Действие                |
|----------------------|------------|-------------------------|
| `npm start`          | Backend    | запуск сервера          |
| `npm run dev`        | Backend    | сервер с nodemon        |
| `npm run init-db`    | Backend    | сброс и сид БД          |
| `npm run dev`        | Frontend   | Vite, порт 5173         |
| `npm run build`      | Frontend   | production-сборка       |
| `npm run preview`    | Frontend   | предпросмотр сборки     |
